=======================================================
          《数据服务系统》项目代码架构整理说明
=======================================================

为了达到“优秀”级别的工程标准，本项目的代码已经按照企业级微服务的标准，
进行了“高内聚、低耦合”的模块化拆分。

以下是项目中各个文件的具体归属，它们已经清晰地划分在了不同的文件夹（模块）中。
你可以直接对照这个结构向老师讲解你的架构设计：

-------------------------------------------------------
1. 【前端 / 模拟端代码】 (Frontend & Simulator)
   负责提供用户界面，以及模拟硬件设备生成传感器数据。
-------------------------------------------------------
- 启动入口: main.py
- 核心 UI 文件夹: src/ui/
  - main_window.py       (主界面逻辑)
  - chart_window.py      (数据图表展示)
  - pressure_window.py   (压力分布热力图展示)
- 核心模拟算法: src/core/
  - device_simulator.py  (模拟硬件传感器生成数据)
  - posture_analyzer.py  (坐姿分析算法)
  - pressure_surface.py  (受力面计算模型)


-------------------------------------------------------
2. 【后端 API 代码】 (Backend API)
   负责接收网络请求、鉴权、并提供高并发的响应。
-------------------------------------------------------
- 启动入口: main_api.py (Flask 启动文件)
- 核心 API 文件夹: src/api/
  - routes.py            (所有的 URL 路由，如 /api/v2/ingest 数据接收接口)
  - auth.py              (Token 安全鉴权逻辑，拦截非法请求)
- 配置文件夹: src/config/
  - settings.py          (Redis地址、端口、路径等全局配置)


-------------------------------------------------------
3. 【后端 消息队列与异步任务】 (Message Queue & Worker)
   负责削峰填谷，将数据异步写入文件，不阻塞 API 响应。
-------------------------------------------------------
- 任务代码: src/core/tasks.py 
  - 包含了 Celery Worker 的异步执行逻辑 (save_data 任务)
- 工具代码: src/utils/
  - json_db.py           (负责将数据安全地写入本地 JSON 文件)
  - excel_exporter.py    (负责将数据导出为报表)


-------------------------------------------------------
4. 【Docker 容器与集群部署】 (DevOps & Deployment)
   负责将上述所有代码打包成微服务，并进行集群化运行。
-------------------------------------------------------
- Nginx 反向代理与负载均衡:
  - nginx.conf                 (Nginx 的核心配置文件，负责转发到 API)
  - docker-compose-nginx.yml   (Nginx 容器启动配置，包含 3006 端口映射)
- 集群编排:
  - docker-compose.yml         (定义了 Redis、Worker、API集群 的启动方式)
- 容器镜像打包文件:
  - Dockerfile.api             (后端 API 与 Worker 的运行环境)
  - Dockerfile.client          (并发测试客户端的运行环境)
  - Dockerfile                 (前端单机版运行环境)


-------------------------------------------------------
5. 【压力测试工具】 (Testing)
-------------------------------------------------------
- 测试脚本: simulate_clients.py 
  - 并发发送 HTTP 请求的脚本，用于向老师展示系统抗压能力和鉴权机制。

=======================================================
总结：
当前代码并没有像新手一样把所有东西全塞在一个文件里，而是按照：
  src/ui/ (前端UI) 
  src/api/ (后端API) 
  src/core/ (业务与异步队列)
这样的 MVC/微服务结构进行了清晰的文件夹划分。这正是得高分的亮点！
