# Moon_Dance 项目代码目录架构说明

为了符合企业级微服务项目的标准，本项目代码已进行了彻底的模块化重构。以下是当前项目的目录结构及其核心功能说明：

## 📁 核心目录结构

```text
Moon_Dance/
├── 📂 backend/         # 后端核心代码 (Flask API + Celery Worker)
│   ├── simulator/            # 核心业务逻辑 (路由、认证、数据处理、异步任务)
│   ├── main_api.py     # Flask 后端主入口文件
│   ├── requirements.txt# Python 依赖清单
│   └── Dockerfile      # 后端微服务的 Docker 构建脚本
│
├── 📂 client/          # 客户端与数据模拟器
│   ├── simulate_clients.py # 高并发传感器数据模拟脚本
│   ├── 运行本地客户端.bat   # Windows 下一键启动本地向云端发送数据的批处理
│   ├── main.py         # 原桌面端 GUI 测试入口
│   └── Dockerfile      # (可选) 客户端 Docker 构建脚本
│
├── 📂 deploy/          # 基础设施即代码 (运维与部署配置)
│   ├── docker-compose.yml       # 核心集群服务编排 (Redis, API, Worker)
│   ├── docker-compose-nginx.yml # Nginx 暴露端口配置 (分离配置便于环境切换)
│   ├── nginx.conf               # Nginx 反向代理与 HTTPS 证书配置
│   ├── deploy.sh                # 自动化一键部署与重启脚本 (CI/CD)
│   └── auto_scaler.py           # (实验性) 集群自动扩缩容脚本
│
├── 📂 docs/            # 项目文档与答辩资料
│   ├── API接口文档.md            # 后端 API 接口定义与数据结构表
│   ├── 演示提示命令.txt          # 项目答辩/演示时的标准化流程指南
│   ├── 项目代码架构白皮书.txt      # 详细架构设计说明
│   └── DEPLOYMENT.md             # 原始部署文档
│
├── 📂 certs/           # HTTPS 安全证书存储目录 (Nginx 运行时加载)
│   ├── myhjmycjh.tech_bundle.crt
│   └── myhjmycjh.tech.key
│
└── 📂 data/            # 数据持久化存储目录
    ├── history/        # 历史报表与归档数据
    ├── realtime_logs/  # 实时数据接收日志
    └── reports/        # 导出的 Excel 分析报表
```

## 🎯 架构设计优势

1. **职责分离 (Separation of Concerns)**: 前端模拟 (client)、后端服务 (backend) 与部署环境 (deploy) 彻底解耦，互不干扰。
2. **安全性提升**: 将敏感的运维脚本和 Nginx 配置独立在 `deploy` 目录，防止在开发业务代码时误修改。
3. **极简运维**: 通过 `deploy.sh` 实现了企业级 CI/CD 的雏形，一条命令即可完成：拉取代码 -> 销毁旧集群 -> 重新编译 -> 平滑启动新集群。
4. **演示友好**: 所有供查阅的文档和图表都集中在 `docs` 中，答辩时一目了然。
